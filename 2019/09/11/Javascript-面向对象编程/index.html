Javascript 面向对象编程

  • 关于JavaScript

    Javascript是一种基于对象（object-based）的语言，你遇到的所有东西几乎都是对象。但是，它又不是一种真正的面向对象编程（OOP）语言，因为它的语法中没有class（类）。

  • 封装

    1. 构造函数模式

    为了解决从原型对象生成实例的问题，Javascript提供了一个构造函数（Constructor）模式。所谓”构造函数”，其实就是一个普通函数，但是内部使用了this变量。对构造函数使用new运算符，就能生成实例，并且this变量会绑定在实例对象上。
    function Person(name,color){
    this.name=name;
    this.color=color;
    }
    缺点:构造函数模式对于想要共享的属性和方法，会在实例上重新复制一遍，造成内存浪费

    2. 构造函数模式的验证

    由构造函数创建的实例对象，含有一个constructor属性，指向构造函数
    实例可以通过运算符instanceof ，验证原型对象与实例对象之间的关系

    3. Prototype模式

    Javascript规定，每一个构造函数都有一个prototype属性，指向另一个prototype对象。这个对象的所有属性和方法，都会被构造函数的实例继承。

    在构造函数模式的基础上，把共享的不变的属性和方法，直接定义在prototype对象上。
    实例中的这些属性和方法都是同一个内存地址，指向prototype对象，提高了运行效率。
    Person.prototype.share = " "

    4. Prototype模式的验证方法

    • isPrototypeOf()

      判断 proptotype对象和某个实例之间的关系。

    • hasOwnProperty()

      判断某一个属性到底是本地属性，还是继承自prototype对象的属性。

    • in运算符

      判断，某个实例是否含有某个属性，不管是不是本地属性。

  • 继承

    prototype模式

    让A继承B
    A.prototype = new B();把A的prototype对象，指向一个B的实例
    A.prototype.constructor = A; 把A.prototype对象的constructor值改为A。
